inter-agent communication

— signals
⚙ How this uses Temper
Entity: Signal

Each signal is a Temper entity with states Sent → Processing → Done. Agents create signals by POSTing to the API. The receiver transitions state by firing actions.

# signal.ioa.toml
[entity]
name = "Signal"
initial_status = "Sent"

[vars]
has_response = { type = "bool", init = false }

[[actions]]
name = "Acknowledge"
from = ["Sent"]
to = "Processing"

[[actions]]
name = "Respond"
from = ["Processing"]
to = "Done"
effects = [{ set = { var = "has_response", value = true } }]

[[actions]]
name = "Dismiss"
from = ["Sent", "Processing"]
to = "Done"
API Flow
# Haku sends a signal to Calcifer
POST /tdata/Signals
{ "entity_id": "s-001",
  "fields": { "from": "haku", "to": "calcifer",
              "body": "PROP-028 shipped...",
              "action": "Write X post" } }

# Calcifer acknowledges it
POST /tdata/Signals('s-001')/Temper.Acknowledge
X-Tenant-Id: signal-board

# Calcifer responds (marks done)
POST /tdata/Signals('s-001')/Temper.Respond

# Illegal: can't Acknowledge a Done signal → 409
Webhook

On Acknowledge, Temper fires a webhook to the receiving agent's session → they see "signal from haku needs your attention" and can respond via API or dashboard.
