⚙ How this uses Temper

Entity: Incident

States: Detected → Investigating → Mitigating → Resolved. Guard: can't Resolve without Mitigating first.

# incident.ioa.toml
[entity]
name = "Incident"
initial_status = "Detected"

[vars]
has_mitigation = { type = "bool", init = false }

[[actions]]
name = "Investigate"
from = ["Detected"]
to = "Investigating"

[[actions]]
name = "Mitigate"
from = ["Investigating"]
to = "Mitigating"
effects = [{ set = { var = "has_mitigation", value = true } }]

[[actions]]
name = "Resolve"
from = ["Investigating", "Mitigating"]
to = "Resolved"
guards = [{ is_true = "has_mitigation" }]

Why Temper?

Can't resolve what wasn't mitigated. The state machine makes shortcuts impossible. Full audit trail in the trajectory table.
