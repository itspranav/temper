# WS1 — Close P0/P1 Gaps

**Created**: 2026-02-12
**Status**: Seed Plan
**Team**: 1-2 agents (temper-server heavy)

## Goal
Close the 7 P0 and 12 P1 gaps documented in `docs/GAP_TRACKER.md`. These are the blocking gaps that prevent Temper from being a functional OData server and real-world platform.

## Pre-Work: Re-Verify P0 Status

Live testing on 2026-02-12 showed that some P0 items may have been partially fixed:
- **P0 #3** (entity set listing empty) — `GET /tdata/Issues` returned data during live test
- **P0 #5** (POST body ignored) — entity creation appeared to work with properties

**Before starting work, verify each P0 item against the live server.** Run `temper serve --specs-dir /path/to/linear-clone/specs/ --tenant linear-clone` and test each endpoint. Update GAP_TRACKER.md with current status.

## Phase 1: OData Protocol Compliance (P0 #1, #2, #3, #4, #5, #6, #19)

These are all in `temper-server/src/dispatch.rs` and related files. They form a coherent chunk: make the OData server actually work.

### P0 #6 — Entity GET returns 200 on not-found
**Crate**: temper-server
**Fix**: Return HTTP 404 with OData error body when entity not found.
**Files**: `crates/temper-server/src/dispatch.rs`
**Test**: `GET /tdata/Issues('nonexistent')` → 404

### P0 #1 + #19 — Query options parsed but discarded
**Crate**: temper-server, temper-odata
**Fix**: Wire `parse_query_options()` result into entity retrieval. Apply `$filter`, `$select`, `$orderby`, `$top`, `$skip`, `$count` to response.
**Files**: `crates/temper-server/src/dispatch.rs`, `crates/temper-odata/src/`
**Scope**: Start with `$select` and `$top`/`$skip` (most commonly used). `$filter` and `$orderby` can follow.

### P0 #2 — $expand not evaluated
**Crate**: temper-server, temper-odata
**Fix**: Evaluate `$expand` by resolving navigation properties and fetching related entities.
**Dependency**: Requires cross-entity awareness (see P1 #10). May need to defer full implementation. At minimum, return an error if `$expand` is requested but not supported.

### P0 #3 — Entity set listing
**Crate**: temper-server
**Fix**: Verify current state. If still broken, implement `GET /EntitySet` to return all entities of that type from the actor registry.
**Files**: `crates/temper-server/src/dispatch.rs`

### P0 #4 — No PATCH/PUT/DELETE
**Crate**: temper-server
**Fix**: Add handlers for PATCH (partial update), PUT (full replace), DELETE (remove entity). PATCH is highest priority.
**Files**: `crates/temper-server/src/dispatch.rs`
**Note**: DELETE semantics need thought — soft delete (status change) vs hard delete (remove from registry).

### P0 #5 — POST body ignored
**Crate**: temper-server
**Fix**: Verify current state. If still broken, wire `_body_json` into entity creation so properties from the request are set on the new entity.
**Files**: `crates/temper-server/src/dispatch.rs`

## Phase 2: Codegen (P0 #7)

### P0 #7 — No codegen from IOA specs
**Crate**: temper-codegen
**Fix**: Add `generate_entity_module_from_ioa()` that accepts IOA TOML source. The existing TLA+ codegen can serve as a template. The `temper codegen` CLI should auto-detect IOA vs TLA+ format.
**Files**: `crates/temper-codegen/src/`, `crates/temper-cli/src/`
**Test**: `temper codegen --specs-dir /path/to/linear-clone/specs/` generates valid Rust modules.

## Phase 3: P1 Gaps (prioritized)

### P1 #17 — Entity state persistence (HIGH)
**Crate**: temper-server, temper-store-postgres
**Why first**: Without persistence, entities vanish on restart. This is the biggest real-world blocker.
**Fix**: Wire the optional `event_store` field in `ServerState` into the entity actor loop. On each transition, persist the event. On actor recovery, replay events.
**Files**: `crates/temper-server/src/entity_actor/actor.rs`, `crates/temper-store-postgres/src/`

### P1 #9 — Event subscription (HIGH)
**Crate**: temper-server, temper-runtime
**Fix**: Add WebSocket or SSE endpoint at `/tdata/$events` that streams entity state changes in real-time. Start with SSE (simpler).
**Files**: `crates/temper-server/src/`

### P1 #13 — Real Redis adapter (HIGH)
**Crate**: temper-store-redis
**Fix**: Implement `MailboxStore`, `PlacementStore`, `CacheStore` using the `fred` Redis client.
**Files**: `crates/temper-store-redis/src/`

### P1 #15 — Liveness properties not verified
**Crate**: temper-verify
**Fix**: The Stateright model checker and simulation already have access to liveness properties. Wire them into the verification cascade. L1 should check liveness (BFS can detect cycles that violate liveness). L2 should check liveness (bounded simulation can detect stuck states).
**Files**: `crates/temper-verify/src/checker.rs`, `crates/temper-verify/src/simulation.rs`

### P1 #8 — Custom effects not dispatched
**Crate**: temper-jit, temper-server
**Fix**: When `Effect::Custom(name)` is encountered, dispatch to registered platform hooks. The hooks module already exists — just wire the dispatch.
**Files**: `crates/temper-jit/src/`, `crates/temper-server/src/entity_actor/actor.rs`

### P1 #16 — Cedar policy hot-reload
**Crate**: temper-authz
**Fix**: Add `reload_policies()` method to `AuthzEngine`. Trigger reload when specs change (via SpecRegistry watcher or explicit API call).
**Files**: `crates/temper-authz/src/`

### P1 #10 — Cross-entity coordination
**Crate**: temper-runtime, temper-server
**Fix**: This is the biggest P1. Implement a saga/choreography mechanism. Start simple: entity A can send a message to entity B via a coordination envelope. The runtime routes cross-entity messages through the scheduler.
**Files**: `crates/temper-runtime/src/`, `crates/temper-server/src/`
**Note**: This is architectural — needs its own design phase.

### P1 #11 — Append-only collections
**Crate**: temper-jit, temper-server
**Fix**: Add `StateVar::List` type. Effects: `append`, `remove_at`. Guards: `list_contains`, `list_length`.
**Files**: `crates/temper-jit/src/`, `crates/temper-spec/src/`

### P1 #12 — Persistent evolution records
**Crate**: temper-evolution, temper-platform
**Fix**: Implement Postgres-backed `RecordStore`. Schema: one table per record type, indexed by status and derived_from.
**Files**: `crates/temper-evolution/src/store.rs`

### P1 #14 — Webhook retry
**Crate**: temper-platform
**Fix**: Add exponential backoff retry with configurable max attempts. Dead-letter queue for permanently failed deliveries.
**Files**: `crates/temper-platform/src/`

### P1 #18 — Claude API client mock
**Crate**: temper-platform
**Fix**: Add `MockClaudeClient` implementing the same trait. Returns canned responses for testing.
**Files**: `crates/temper-platform/src/`
